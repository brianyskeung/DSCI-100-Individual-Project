1. Data Description:

There are two datasets to examine as part of the project. The first dataset "players.csv" contains player-specific information and cumulative play data, with 196 rows and 7 columns. The second dataset is "sessions.csv" which contains information about each individual session played, with 1535 rows and 5 columns.

players.csv

Variables/Columns

  1. experience (chr): A categorical variable used to report the player's experience level ("Beginner", "Amateur", "Regular", "Veteran", "Pro").
  2. subscribe (lgl): A boolean categorical variable used to represent whether the players are subscribed or not subscribed.
  3. hashedEmail (chr): A categorical variable used to recognize each player. Each player has a unique code to identify them by.
  4. played_hours (dbl): A numerical variable used to report the player's total hours played.
  5. name (chr): A categorical variable used to identify the player's name.
  6. name (chr): A categorical variable used to identify the player's gender.
  7. Age (dbl): A numerical variable used to report the player's age.

sessions.csv

Variables/Columns

  1. hashedEmail (chr): A categorical variable used to recognize each player. Each player has a unique code to identify them by.
  2. start_time (chr): The date and time of each session start stored as a character.
  3. end_time (chr): The date and time of each session end stored as a character.
  4. original_start_time (dbl): A numerical variable used as a Unix time identifier to represent the start time.
  5. original_end_time (dbl): A numerical variable used as a Unix time identifier to represent the end time.

Issues with the data include the presence of 2 NA's in quantitative variables like Age and end_time, which should be addressed either with mean imputation or removing them. Another issue may be that there is a discrepancy in the proportions of players for each category. There are significantly more Amateurs (63) and very little Pros (14) compared to other categories in the dataset which may skew the results. The distribution of Age and Playtime is also very skewed.

2. Questions:

Broad question to address: We would like to know which "kinds" of players are most likely to contribute a large amount of data so that we can target those players in our recruiting efforts.

Specific question to address: Can a player's age and experience predict their total playtime in the dataset?

The data can be used to address these questions as we have identified the exploratory/response variable as the total playtime, and the predictors as the age or skill level. Total playtime serves as the measurement of each player's contribution, whereas age and skill level/experience gives us an image of the demographics to recruit from.

4. Methods and Plan

My proposed method would be to use multivariable linear regression. This method is appropriate because it allows us to see how multiple predictors (in this case Age and Experience) predict a continous variable (total playtime). Linear regression is useful because it provides an interpretable math equation that can be used to explain how each predictor affects the reponses, which we can then use to identify which combination/demographic of participants would contribute the most data. However, multivariable linear regression comes with several limitations. One limitation is that the model assumes a linear relationship between each predictor and the response, which may not always be valid. If the true relationship is nonlinear, the model may perform poorly. Another weakness is the sensitivity to outliers, which can disproportionately influence the regression line and distort the results. Also, if the predictors are highly correlated then the estimates may not be as accurate either. To compare and select the best model, I will use 5-fold cross-validation on the training set. This will allow me to evaluate how well the regression model generalizes across different subsets of the data. Model performance will be assessed using metrics such as RMSPE which measures predictive accuracy on unseen data. After cross-validation, I will select the model that produces the lowest average error. The final chosen model will then be evaluated on the test set to validate its performance.

Process

First, I will split the cleaned dataset into a training set (80%) and a test set (20%). After splitting, I will create a modeling recipe with response variable as played_hours and predictors as age and experience. I wil also handle missing values via mean imputation.

Next, I will specify a linear regression model and combine it with the recipe in a workflow. I will then perform 5-fold cross-validation on the training data and tune the workflow if necessary. After identifying the best-performing model based on RMSPE, I will evaluate it on the test set and report performance metrics such as RMSE to provide a full picture of how well the predictors explain total playtime.
